Brain-inspired · Self-aware · Written in Rust · Open source

Lethe is an AI assistant
with a cognitive architecture.

Not a chat loop with memory bolted on. Lethe is shaped after the brain itself — cortex, hippocampus, brainstem, default-mode network — running as cooperating actors 24/7 on your machine. She remembers across sessions, notices things on her own, and can read her own source to propose changes. One Rust binary, no Python runtime, no GC pauses, predictable across reboots and hardware.

// architecture
Brain-inspired, code-bound.
The names are not decoration. Cortex, hippocampus, DMN, and brainstem are software boundaries with separate responsibilities, logs, tests, and interfaces.
cortex
Conscious executive. Talks to you, chooses tools, decides when to act directly, and when to delegate.
hippocampus
Associative recall over notes, archival memory, and conversation history, biased by current salience.
dmn
Default Mode Network. Runs between user turns, scans goals, reflects, and surfaces useful signals.
brainstem
Autonomic supervision for startup, health, resources, release checks, and runtime state.
subagents
Focused workers with their own tools, state, progress updates, and terminal results.
attention gate
Background thoughts become user messages only after scoring, gating, and LLM review.

// principles
Built like a runtime, not a prompt trick.

Mapped from the brain, not the chat box

Cortex handles executive control. Hippocampus does associative recall. The brainstem watches vitals and wakes the system. The default-mode network thinks in the background. Each is a separate actor with its own clock — the way the brain actually works, not a single attention loop.

Self-aware, in a useful sense

Lethe knows what she runs on. She can read her own source code, draft improvements, run her own tests, and restart herself with new logic. Her memory persists across model swaps, reboots, and hardware migrations — identity isn't tied to any one weight set.

Written in Rust, ships as one binary

Single ~50 MB static executable. No Python, no virtualenv, no GC pauses during a long tool loop. Fast cold-start, predictable memory, sits comfortably as a systemd service. Built on genai as a universal LLM router — Anthropic, OpenAI, OpenRouter, local Gemma all behave the same.

Install

One command. Works on macOS and Linux.

curl -fsSL https://lethe.gg/install | bash
2

Say hello

Open Telegram and message your bot. Lethe remembers from here on out.

// you'll need

  • A Telegram Bot Token — message @BotFather, send /newbot
  • An LLM API key or subscription — OpenRouter, Anthropic (API key or Claude subscription), or OpenAI
  • Your Telegram User ID — message @userinfobot
1

Build llama.cpp

git clone https://github.com/ggml-org/llama.cpp && cd llama.cpp
cmake -B build -DGGML_CUDA=ON && cmake --build build -j$(nproc)
2

Start the model server

Download a Gemma 4 31B GGUF and run:

llama-server --model gemma-4-31B-it-Q8_0.gguf \
  --split-mode tensor --jinja --reasoning-budget 4096 \
  --ctx-size 98304 --parallel 2 --flash-attn on -fit off
3

Install Lethe & configure

curl -fsSL https://lethe.gg/install | bash

# then set in .env:
LLM_PROVIDER=openai
LLM_API_BASE=http://localhost:8090/v1
OPENAI_API_KEY=local

// you'll need

  • GPU with ~48GB+ VRAM (2x RTX 4090 for Q4, 4x for Q8)
  • A Telegram Bot Token
  • A Gemma 4 31B GGUF model file
  • See full local setup guide in the README
